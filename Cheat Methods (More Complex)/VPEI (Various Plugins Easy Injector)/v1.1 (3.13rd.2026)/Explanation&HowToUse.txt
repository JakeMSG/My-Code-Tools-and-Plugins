================ Explanation
==== Another quick Batch-powered utility to help you quickly add Mods to any MV game using the MV Mod Loader, but also being able to inject it into any "plugin.js" file without having to modify the file manually yourself
== Link to the MV Mod Loader source: https://github.com/1d51/mv-mod-loader
==== I made this using the knowledge from making ERDO, but with slight improvements 
==== At its core, it's a simple batch file that runs commands on the "Xchang32" CMD utility 
== Link to the Website of the Utility (pack of Utilities): http://bbses.info/mirrors/clayruth.com/index.html


================ How to Use
======== I. Injection of Plugins
==== 1. Copy the files from here and move them in the folder of your RPG Maker game (right next to it's Executable (".exe") file)
==== 2. Run the "[Run This!]" file (the batch file)
==== 3. Now, simply select the desired option, and Exit when done (either via the option or by closing the window)
== When it adds the plugin, it adds it at the very bottom of the list
== When it removes the plugin, it removes it from wherever first on the list it finds it (if there's multiple entries of it in the plugins.js (for some reason) you'll have to use the remove option multiple times)
======== Possible Issue - Long paths
==== If it shows that it "Cannot Change", it's very likely that your current folder path is too long
== A quick side-fix is to temporarily move the game folder closer to your root path (eg: on "F:\" instead of "F:\[VERY_LOOOOOOOOOOOOOOOOOOOOOOOOOOOOOOOOONG__PAAAAAAAAAAAAAAAAATH]"), run the utility, then move the folder back




================ Advanced Use - Editing the "[Run this!].bat" (to search for custom Strings)
======== For the ".bat" (Cmd) commands
==== Look it up on the internet
======== For the "xchange32"-specific commands
==== To search, within your string:
== For a literal "," or "^" signs => double them (eg: ",," and "^^")
== For ASCII characters => Use "^" sign, then their code next to it
= "\n" (Line Feed / New Line) => "^10"
= Quotes (" " ") => "^34"
= "%" => "^37"
= ";" => "^59"

= "[" => "^91"
= "]" => "^93"
= "{" => "^123"
= "}" => "^125"

= "0" => "^48"
= "1" => "^49"
= "9" => "^57"
== If you have another number next to the ASCII character you want to replace with their code, it will end up being taken by the code (eg: replacing "[1" with "^911"), so you'll instead also have to write the number next to that code as its ASCII code (eg: to properly replace "[1", you'll have to use "^91^49")

========= To do Replacements for longer strings
==== First, get the string, and do, on it, Regex Search&Replaces separatedly to add the ASCII codes to be able to make that string search-able using the "XChange32" program
==== Now, if you're searching for the long string (>240 chars), break apart the string on smaller parts and use the command repeatedly instead of once for the longer string
== For example, if you need to find and delete a >240 chars, you can easily delete each of its parts; If you instead want to replace a >240 chars, delete parts of it not needed to find it specifically, then for the minimum part needed to find it, use it and replace only with that part
==== Replacement string can be a lot longer with no issues

======== Longer commands in CMD batch file
==== Single-line => Split commands with "&"
==== Multi-line => Surround all the commands to run together, with "()", and split each command on its own line

======== Debug Errors with "Xchange32" commands
==== append this command at the end: "& echo %ERRORLEVEL%"
==== The error levels:
0 = changed, 1 = not changed, 2 = unable to rewrite/del,
3 = syntax or memory error,  4 = file not found, 5 = error during read/write,
6 = insufficient disk space, 7 = read-only file, 8 = write-protected.

======== Current Issues
==== You can't search for multiple consecutive newlines (but you can replace with multiple newlines)
## Instead, try to do separate smaller searches to circumvent this issue
